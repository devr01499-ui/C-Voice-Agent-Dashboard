# Governance Rules

## Engineering Standards
- All environment variables must be prefixed correctly for the environment (e.g., `VITE_` for client-side).
- Production credentials must never be committed to version control.
- All code changes must follow the defined `implementation_plan.md`.

## Decision Protocol
- Every significant technical choice requires an entry in `ARCHITECTURE_DECISIONS.md`.
- Rationale and trade-offs must be explicitly documented.
